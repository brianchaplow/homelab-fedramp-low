# ia-2.8 - \[Identification and Authentication\] Access to Accounts — Replay Resistant

## Control Statement

Implement replay-resistant authentication mechanisms for access to [Selection (one or more): privileged accounts; non-privileged accounts].

## Control Assessment Objective

replay-resistant authentication mechanisms for access to [Selection (one or more): privileged accounts; non-privileged accounts] are implemented.

## Control guidance

Authentication processes resist replay attacks if it is impractical to achieve successful authentications by replaying previous authentication messages. Replay-resistant techniques include protocols that use nonces or challenges such as time synchronous or cryptographic authenticators.

### This System

All authentication mechanisms in use within the Managed SOC Service boundary are replay-resistant, covering both privileged and non-privileged accounts.

SSH key authentication -- used for all in-boundary host access (brisket, haccp, smokehouse, smoker, pitcrew, sear, dojo, regscale) -- uses the SSH protocol's cryptographic challenge-response handshake. Each session uses a unique nonce generated by the server; capturing and replaying a previous session transcript cannot authenticate a new session. Wazuh API JWTs are short-lived (900-second default expiry); each authentication call in `pipelines/common/wazuh.py` issues a fresh token that cannot be reused after expiry. RegScale CE JWTs carry a 24-hour TTL per ADR 0003 and ADR 0006 Deviation 7; every pipeline invocation calls `POST /api/authentication/login` to obtain a fresh token, and the old token expires at its TTL. Tailscale WireGuard establishes ephemeral Diffie-Hellman key exchange per session, producing session keys that are not replayable.

HTTPS service consoles (Wazuh Dashboard, Shuffle SOAR, Velociraptor) protect session tokens in transit via TLS; session cookies are bound to the TLS session. A known partial gap exists for TheHive, Cortex, DefectDojo, and RegScale, which serve on HTTP on the internal lab network -- token transmission over HTTP is technically replayable within the same network segment. This risk is accepted as a lab-posture deviation per ADR 0003 and ADR 0004, mitigated by VLAN isolation and managed-switch ACLs restricting lateral movement.
