# cm-10 - \[Configuration Management\] Software Usage Restrictions

## Control Statement

- \[a.\] Use software and associated documentation in accordance with contract agreements and copyright laws;

- \[b.\] Track the use of software and associated documentation protected by quantity licenses to control copying and distribution; and

- \[c.\] Control and document the use of peer-to-peer file sharing technology to ensure that this capability is not used for the unauthorized distribution, display, performance, or reproduction of copyrighted work.

## Control Assessment Objective

- \[CM-10a.\] software and associated documentation are used in accordance with contract agreements and copyright laws;

- \[CM-10b.\] the use of software and associated documentation protected by quantity licenses is tracked to control copying and distribution;

- \[CM-10c.\] the use of peer-to-peer file sharing technology is controlled and documented to ensure that peer-to-peer file sharing is not used for the unauthorized distribution, display, performance, or reproduction of copyrighted work.

## Control guidance

Software license tracking can be accomplished by manual or automated methods, depending on organizational needs. Examples of contract agreements include software license agreements and non-disclosure agreements.

### This System

The homelab SOC uses only open-source software under documented licenses, with one proprietary exception reviewed before deployment. The deployed software stack and associated licenses are: Wazuh (GPL v2), OpenSearch (Apache 2.0), ELK -- Elasticsearch, Kibana, Logstash, Filebeat (Elastic License 2.0; SSPL restrictions on offering ELK as a managed service to others do not apply as this is a private homelab, not a commercial MSS offering), Shuffle (GPL), Velociraptor (AGPL), Arkime (Apache 2.0), Zeek (BSD), Suricata (GPL), OpenCTI (Apache 2.0; Enterprise Edition tier not used), TheHive (AGPL), Cortex (AGPL), DefectDojo (BSD 3-Clause), and RegScale CE (proprietary; CE license explicitly reviewed and accepted during deployment per ADR 0001 EULA analysis). No quantity-licensed commercial software requiring seat-count tracking is deployed -- all software is either open-source or single-instance free-tier. No peer-to-peer file sharing technology is present on any in-boundary host.

The Elastic ML trial license (activated 2026-03-17, 30-day) is a time-limited evaluation feature whose expiration reduces ML functionality to read-only without affecting core ELK operation; this licensing event is tracked in CLAUDE.md. ADR 0001 documents the formal EULA analysis conducted for RegScale CE prior to deployment -- the only proprietary software requiring explicit license acceptance. CLAUDE.md §Service Inventory provides the canonical software list per host. The primary gap driving `partial` status is the absence of a formal software usage policy document; the licensing posture is sound and documented in ADRs, but no dedicated policy file has been committed.

#### Implementation Status: partial
