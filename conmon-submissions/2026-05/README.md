### What drove the reduction

On April 12, 2026, the operator performed real remediation across all four in-boundary Ubuntu hosts (brisket, haccp, dojo VM, regscale VM):

1. **Stale kernel removal.** Each host had one or more old linux-image packages installed but no longer booted. Brisket had `linux-image-6.8.0-107-generic` (stale; running kernel is `6.17.0-19` via the HWE track). Haccp, dojo, and regscale each had `linux-image-6.8.0-106-generic` (stale; running kernel is `6.8.0-107`). These were removed via `apt remove` (brisket required removing the non-HWE `linux-image-generic` metapackage first, since it held `6.8.0-107` as a dependency). The Wazuh Indexer confirmed: 0 findings for `linux-image-6.8.0-106-generic` post-remediation.

2. **Package upgrades.** `apt upgrade` on all four hosts patched apparmor, Docker CE, fwupd, systemd, linux-firmware, netplan, and several other packages. Haccp also upgraded filebeat from 8.17.10 to 8.19.14. Five unique CVEs were fully remediated and no longer appear in the May scan.

3. **Wazuh agent restart.** After remediation, all four agents were restarted to force an immediate syscollector re-inventory. The vulnerability detector re-evaluated within 5 minutes.

The April cycle's scan evidence (in `../2026-04/scan-evidence/`) serves as the pre-remediation baseline. The May cycle's scan evidence reflects the post-remediation state. The diff is real: no findings were closed by marking them in DefectDojo, and no synthetic findings were created or staged. The Wazuh Indexer is the source of truth, and the reduction comes from packages that are genuinely no longer installed on the in-boundary hosts.

### Remaining findings (4,876)

- **linux-image-6.8.0-107-generic** (2,694 items): still the running kernel on haccp, dojo, and regscale. These are real findings that require a kernel upgrade + reboot to close, which is a planned future remediation.
- **linux-image-6.17.0-19-generic** and **linux-image-6.17.0-20-generic** (898 items each): brisket's current and pending kernels. Findings against these require upstream Ubuntu security patches.
- **Non-kernel packages** (386 items across ~70 packages): lower-severity findings in packages like Qemu-Guest-Agent, Busybox, OpenSSL, Amd64-Microcode, and various libraries. These are candidates for the next remediation pass.

### Smokehouse note

Smokehouse (10.10.20.10) is a QNAP NAS running kernel 5.10.60-qnap. It has no apt-based package management. Its 46 Wazuh findings are QNAP-specific and cannot be remediated via the same process as the Ubuntu hosts. QNAP firmware updates are the remediation path for those findings.
