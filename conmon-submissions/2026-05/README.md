### What drove the reduction

The May 2026 cycle reflects the cumulative effect of two real remediation passes against the April baseline:

1. **2026-04-12 pass.** Removed stale `linux-image-6.8.0-106-generic` from haccp/dojo/regscale and stale `linux-image-6.8.0-107-generic` from brisket. Brought the April 16,944 baseline to a 4,876 mid-cycle state. (The original May submission generated that day captured this snapshot. Its artifacts have been replaced by this run.)

2. **2026-05-04 pass.** All four in-boundary Ubuntu hosts (brisket, haccp, dojo VM, regscale VM) had their HWE kernels upgraded to `linux-image-6.17.0-23-generic` during the CVE-2026-31431 "Copy Fail" sweep on 2026-05-03 (full sweep documented in `../../HomeLab-SOC-v3/migration-status` and `memory/`). On 2026-05-04, an explicit purge pass was applied because `apt autoremove` does not drop the immediate-predecessor kernel by default (it retains it as a boot fallback), and the predecessor kernels were the largest residual contributors to the POA&M. Specifically:

   - `linux-image-6.17.0-22-generic` + matching `linux-modules-6.17.0-22-generic` and `linux-modules-extra-6.17.0-22-generic` purged from all four hosts (3,556 findings dropped: 4 hosts × 889 CVEs/kernel).
   - `linux-image-6.17.0-20-generic` + matching `linux-modules-6.17.0-20-generic`, `linux-modules-extra-6.17.0-20-generic`, `linux-modules-nvidia-535-6.17.0-20-generic`, and `linux-headers-6.17.0-20-generic` purged from haccp (889 findings dropped). This older predecessor had survived yesterday's apt-get upgrade because Ubuntu only retains N-1, not N-2.
   - Orphan dead-weight cleanup on haccp: `nvidia-firmware-535-535.288.01`, `nvidia-kernel-common-535`, `linux-objects-nvidia-535-6.17.0-20-generic`, `libpciaccess0`, `linux-hwe-6.17-headers-6.17.0-20`. No driver loaded on haccp's Quadro P1000.
   - Orphan dead-weight cleanup on brisket: `linux-headers-6.17.0-22-generic`, `linux-tools-6.17.0-22-generic`, `linux-hwe-6.17-tools-6.17.0-22`, `nvidia-firmware-580-580.126.09` (predecessor of yesterday's 580.142 nvidia firmware upgrade).
   - apt-get upgrade on dojo + regscale picked up late-arriving security patches for `curl`, `libcurl4t64`, `libcurl3t64-gnutls`, and `iproute2`.

3. **Wazuh agent restart.** After remediation, all four agents were restarted to force an immediate syscollector re-inventory. The vulnerability detector re-evaluated within 5 minutes; new totals confirmed via `wazuh-states-vulnerabilities-*` aggregation against `agent.name` before the conmon pipeline ran.

The reductions are real: no findings were closed by marking them in DefectDojo, and no synthetic findings were created or staged. The Wazuh Indexer is the source of truth, and the reduction comes from packages that are genuinely no longer installed on the in-boundary hosts.

### Why Critical edged up vs the prior May POAM

The 2026-04-12 May POAM had 20 Critical findings; this cycle has 60. The increase is **not** a regression. The April-12 May POAM was generated against a fleet running `linux-image-6.8.0-107-generic` on 3 hosts and `linux-image-6.17.0-19-generic`+`-20` on brisket. Those kernels each have a different CVE profile than `linux-image-6.17.0-23-generic`, the running HWE kernel on all four in-boundary hosts after the Copy Fail patch. The +40 Critical delta reflects a different kernel's currently-open severity mix, not new exposure.

### Remaining findings (4,195)

- **linux-image-6.17.0-23-generic** (3,556 items, 84.8% of total): the running HWE kernel on all four in-boundary Ubuntu hosts. Findings against this kernel require Canonical to ship a 6.17.0-24 (or later) HWE update plus a coordinated reboot. Not actionable until upstream patches land.
- **Binutils suite** (160 items): `binutils`, `binutils-common`, `binutils-x86-64-linux-gnu`, `libbinutils`, `libctf0`, `libctf-nobfd0`, `libgprofng0`, `libsframe1` -- all 20 items each, all four hosts. Awaiting Ubuntu noble security update.
- **Qemu-Guest-Agent** (24 items): present on dojo + regscale + brisket as VM/hypervisor tooling. Awaiting `qemu-guest-agent` security update.
- **Busybox + OpenSSL FIPS provider + libc** (~80 items): low-severity findings spread across busybox-static, busybox-initramfs, openssl-fips-provider-latest, openssl-libs, libc-bin, libc6, locales. Candidates for next remediation pass.
- **Misc tail** (~70 items): Libelf1t64, Libdw1t64, Libxslt1.1, Libde265-0, patch, python3, python3-libs, tar, vim, libarchive13t64, amd64-microcode. Will close as upstream patches arrive.

### Smokehouse note

Smokehouse (10.10.20.10) is a QNAP NAS running kernel 5.10.60-qnap. It has no apt-based package management. Its 73 Wazuh findings are QNAP-specific and cannot be remediated via the same process as the Ubuntu hosts. QNAP firmware updates are the remediation path for those findings.
