# Runbook — Manual OSCAL Import into RegScale CE

**Scope:** Plan 2 Task 16 fallback.
**Why this exists:** RegScale CE (version deployed 2026-04-08) does
not expose generic OSCAL import endpoints in its Swagger API. The
13 OSCAL-related paths that do exist are all export / validation —
there is no `POST /api/oscal/ssp` or equivalent for
component-definitions or POA&Ms. Until CE adds these endpoints (or
the lab upgrades to RegScale Enterprise), the OSCAL deliverables
generated by `pipelines.sh conmon` have to be loaded via the UI.

**Inputs** (all produced by `pipelines.sh conmon` or its sub-steps):

| Artifact | Path | Source command |
|----------|------|----------------|
| NIST 800-53 Rev 5 catalog | `oscal/catalog/catalog.json` | Plan 2 Task 2 |
| FedRAMP Rev 5 Low profile | `oscal/profile/profile.json` | Plan 2 Task 3 |
| Component-definition | `oscal/component-definition.json` | `pipelines.sh inventory` |
| System Security Plan | `oscal/ssp.json` | `pipelines.sh ssp-assemble` |
| POA&M | `oscal/poam.json` | `pipelines.sh build-poam` |

## Prerequisites

1. RegScale CE must be running and reachable. Verify with
   `./pipelines.sh smoke-regscale` — expect `OK`.
2. Operator credentials (`REGSCALE_USERNAME` / `REGSCALE_PASSWORD`)
   must be valid. These live in `/c/Projects/.env`.
3. The pipeline artifacts must already be generated. Run
   `./pipelines.sh conmon` or the individual commands above first.

## Import Order

OSCAL artifacts reference each other by UUID. Import in dependency
order so RegScale can resolve cross-references as each new artifact
lands:

1. **Catalog** (`oscal/catalog/catalog.json`)
2. **Profile** (`oscal/profile/profile.json`) — references the
   catalog
3. **Component-definition** (`oscal/component-definition.json`) —
   references the profile
4. **System Security Plan** (`oscal/ssp.json`) — references the
   profile and component-definition
5. **POA&M** (`oscal/poam.json`) — references the SSP

## Step-by-step

### 1. Log in

Browse to `http://10.10.30.28/` (or the value of `REGSCALE_URL` in
`/c/Projects/.env`). Authenticate as the admin user defined by
`REGSCALE_USERNAME`.

### 2. Import the catalog

1. Navigate to **Catalogs** in the left sidebar.
2. Click **Create** → **Import from File**.
3. Select `C:\Projects\homelab-fedramp-low\oscal\catalog\catalog.json`.
4. Confirm the source is OSCAL JSON.
5. Wait for the import spinner to finish (~30s for 800-53 Rev 5).

### 3. Import the profile

1. Navigate to **Profiles** (or **Baselines**, depending on the CE
   menu label).
2. **Create** → **Import OSCAL Profile**.
3. Select
   `C:\Projects\homelab-fedramp-low\oscal\profile\profile.json`.
4. The FedRAMP Rev 5 Low baseline should resolve against the catalog
   imported in step 2.

### 4. Import the component-definition

1. Navigate to **Components** → **Component Definitions**.
2. **Create** → **Import OSCAL Component Definition**.
3. Select
   `C:\Projects\homelab-fedramp-low\oscal\component-definition.json`.
4. All seven in-boundary components (brisket, haccp, smokehouse,
   dojo, regscale, opnsense, mokerlink) should appear under the new
   component-definition.

### 5. Import the SSP

1. Navigate to **Security Plans**.
2. **Create** → **Import OSCAL SSP**.
3. Select `C:\Projects\homelab-fedramp-low\oscal\ssp.json`.
4. This is the Plan 2 skeleton — control implementation prose is
   `REPLACE_ME` placeholders until Plan 3 fills them. The SSP still
   loads cleanly; RegScale will display all 156 controls as
   unfinished implementations.

### 6. Import the POA&M

1. Navigate to **POA&M** (or **Issues** → **POA&M Items**).
2. **Create** → **Import OSCAL POA&M**.
3. Select `C:\Projects\homelab-fedramp-low\oscal\poam.json`.
4. The file is ~16 MB with ~8,500 open items for the homelab ConMon
   snapshot. Upload can take 2–5 minutes.

### 7. Verify

Return to **Security Plans**, open the MSS SSP, and confirm the
sidebar shows:

- 156 control implementations
- 7 components (brisket, haccp, smokehouse, dojo, regscale,
  opnsense, mokerlink)
- ~8,500 linked POA&M items

If any of those counts are zero, the import for that artifact
failed — re-run that specific step.

## When this runbook goes away

If a future RegScale CE release ships generic OSCAL import endpoints
(e.g. `POST /api/oscal/ssp`), populate
`pipelines/push/regscale.py::OSCAL_IMPORT_PATHS` with the real paths
and the push pipeline will take over automatically. Delete the
manual step references in Plan 2 Task 16 and update ADR 0006 with
the new deviation resolution.

Until then, this document is the shippable fallback and is the
single source of truth for the Plan 2 demo narrative.
